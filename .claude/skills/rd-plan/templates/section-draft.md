### 작성 규칙

1. **격식체**: 합니다체/입니다체 일관 사용
2. **정량 근거**: 모든 주장에 출처(URL+날짜) 또는 수치 근거
3. **평가 키워드**: 평가기준의 핵심 키워드를 자연스럽게 반영
4. **논리 구조**: 주장 → 근거 → 시사점 흐름
5. **시각화 마커**: 다이어그램/차트가 필요한 위치에 마커 삽입
   ```
   <!-- VISUAL: type={type}, caption="{caption}" -->
   ```

### 금지 표현

- "혁신적인", "획기적인", "최첨단" → 근거 없이 사용 금지
- "다양한", "풍부한", "체계적인" → 구체 수치/사례로 대체
- "~할 수 있습니다" → 정량 근거 제시 후만 허용
- "~것으로 기대됩니다" → 벤치마크/사례 기반 추정으로 대체

### 시각화 생성 규칙

**Library-First**: 시각화 마커 삽입 전, `/library-search {유형} {캡션 키워드}`를 먼저 호출한다.
- 50%+ 매칭 → 기존 에셋 재사용 (경로 참조)
- 매칭 없음 → VISUAL 마커 삽입

**마크다운으로 충분한 유형은 마커 불필요**:
- `comparison` (비교표) → 마크다운 테이블로 작성
- `org` (조직도) → 마크다운 리스트로 작성

### 시각화 마커 유형 (MCP 생성 필요한 것만)

| type | 설명 | 예시 |
|------|------|------|
| architecture | 시스템 구조도 (15+노드) | 전체 아키텍처 |
| flow | 워크플로우 (≤15노드) | 연구 절차 |
| chart | 데이터 차트 | TAM/SAM/SOM |
| concept | 개념도/일러스트 | 핵심 개념 |
| timeline | 타임라인/WBS | 개발 일정 |
| mockup | UI 목업 | PoC 화면 설계 |

### 출력 형식

마크다운으로 작성. 제목은 `###`부터 시작 (상위 구조와 병합 시 레벨 조정).
