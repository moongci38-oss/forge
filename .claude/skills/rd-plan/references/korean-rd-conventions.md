# 한국 R&D 정부과제 사업계획서 작성 관례

## 문체

- **합니다체/입니다체** 일관 (혼용 금지)
- 주어 생략 가능하나 논리적 주어가 명확해야 함
- 수동태 최소화 ("개발되어지다" → "개발한다")

## 구조

- **주장 → 근거 → 시사점** 3단 논리
- 각 섹션 첫 문장에 핵심 메시지 (두괄식)
- 정량 데이터는 출처(기관명, 발행연도) 필수
- 표/그림 번호는 `[표 1]`, `[그림 1]` 형식

## 평가위원 관점

- 평가위원은 **키워드 매칭**으로 빠르게 스캔함
- 평가기준의 키워드를 섹션 제목/첫 줄에 배치
- "왜 이 기술인가", "어떻게 검증하는가", "얼마나 현실적인가" 3가지 질문에 답해야 함
- 30점 배점 섹션에 1페이지 쓰면 감점, 10점 섹션에 3페이지 쓰면 비효율

## 금지

- 근거 없는 수식어: "혁신적", "획기적", "세계 최초" → 특허/논문/벤치마크 근거 필수
- AI 생성 특유 표현: "다양한 분야에서 활용 가능한", "풍부한 경험을 바탕으로"
- 과도한 약어: 첫 등장 시 풀네임(약어) 형태, 이후 약어만
- 비정량적 목표: "성능을 향상시킨다" → "처리 속도를 50ms 이내로 단축한다"

## 시각 자료

- 모든 그림/표에 번호 + 캡션 필수
- 다이어그램은 한글 라벨 (영어 병기 가능)
- 색상 대비 명확 (흑백 인쇄 대비)
- 스크린샷보다 도식 선호

## 분량 배분 공식

```
섹션_페이지 = 전체_페이지 × (해당_평가_배점 / 총_배점)
```

예: 전체 7p, 사업화 30점/100점 → 사업화 = 7 × 0.3 = 2.1p

## HWP 제출 시 주의

- 보안 문서/비밀번호 설정 금지 (PDF 변환 오류)
- PDF 파일로 제출 금지 (HWP만 인정)
- 파일명 규칙: `(과제명 앞 두 단어)_(주관기관명).hwp`
- 대표자 서명 + 기관 직인 필수
