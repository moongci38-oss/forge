# 평가항목 키워드 매핑 (공통)

> 한국 정부 R&D 과제 평가기준에서 자주 사용하는 키워드와 대응 전략.
> 섹션 작성 시 해당 키워드를 자연스럽게 반영해야 고득점.

## 공통 평가 키워드

### 1. 기획 목표 / 혁신성 (통상 20~25점)

| 평가 키워드 | 작성 전략 | 예시 표현 |
|-----------|---------|---------|
| 차별성 | 선행기술 대비 1:1 비교표 | "기존 기술 대비 3가지 차별점" |
| 혁신성 | 특허/논문 기반 신규성 | "선행특허 6건 검토 결과 미확인" |
| 문제 정의 | 정량 데이터로 문제 크기 제시 | "시장의 66%가 해당 문제 경험" |
| 해결 필요성 | Why now? 시급성 | "규제 변화/시장 전환 트렌드" |
| 연구 질문 | 명확한 RQ 제시 | "본 연구가 답하려는 4가지 질문" |

### 2. PoC/PoV 검증 (통상 20~25점)

| 평가 키워드 | 작성 전략 | 예시 표현 |
|-----------|---------|---------|
| 기술 검증 | PoC 시나리오 구체성 | "정확도 >90%, 응답시간 <50ms" |
| 시장 검증 | PoV 계획 타당성 | "해외 팬 N>100, 전환의향 >40%" |
| 실현 가능성 | 기존 기술 기반 | "선행연구 결과 활용 가능" |
| 검증 설계 | 방법론 명시 | "A/B 테스트, 코호트 분석" |

### 3. 수행역량 (통상 15~20점)

| 평가 키워드 | 작성 전략 | 예시 표현 |
|-----------|---------|---------|
| 전문성 | 팀 이력과 문제의 연관성 | "관련 분야 N년 경험" |
| 추진체계 | WBS + 마일스톤 | "월별 산출물 명시" |
| 인력 적정성 | 역할 배분 명확 | "NLP 전문가 + 프론트엔드 + PM" |
| 수행이력 | 기존 정부과제 실적 | "디딤돌/초창패 수행 중" |

### 4. R&D 고도화 / 사업화 (통상 25~30점)

| 평가 키워드 | 작성 전략 | 예시 표현 |
|-----------|---------|---------|
| 비즈니스 모델 | 수익 구조 구체적 | "B2B SaaS + 수수료 50%" |
| 경제적 성과 | 3~5년 매출 전망 | "2028년 흑자 전환" |
| 글로벌 | 해외 진출 전략 | "영미→일본→동남아 순차" |
| 민간투자 | VC 유치 전략 | "AI+K-콘텐츠+특허 = VC 매력" |
| 성장성 | TAM/SAM/SOM | "글로벌 팬덤 경제 TAM 5조" |

## Prefab Visual Library 활용

시각화 생성 시 `/library-search` 스킬로 기존 에셋 먼저 탐색:
- 경로: `~/prefab-visual-library/`
- 재활용 가능: 아이콘, 다이어그램 템플릿, 차트 스타일
- 완전매칭 → 그대로 사용 / 부분매칭 → 수정 후 사용 / 없음 → MCP 생성

## docs-visualization.md 규칙 연동

시각화 유형 판별 시 `~/.claude/rules/docs-visualization.md` 규칙을 따름:
- 게임 로직 → `/game-logic-visualize`
- 인터랙티브 탐색 → `playground`

- 일반 플로우 (≤15노드) → Mermaid 인라인
