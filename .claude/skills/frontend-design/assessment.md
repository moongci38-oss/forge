# Assessment: frontend-design

## 테스트 입력

- input_1: "Design a modern pricing page with 3 tiers: Free, Pro, Enterprise"
- input_2: "Create a dashboard header with user avatar, notifications bell, and search bar"
- input_3: "Build a testimonial carousel component with customer photos and quotes"

## 평가 기준 (Yes/No)

1. 산출물 존재: HTML 파일 저장 경로가 명시되어 있거나, 코드 블록에 HTML/React 코드가 포함되어 있는가?
2. 디자인 설명 포함: 색상, 폰트, 레이아웃 등 시각 디자인 요소가 출력에 2개 이상 구체적으로 설명되어 있는가?
3. 입력 요소 반영: 입력에서 요청한 핵심 요소(티어명, UI 컴포넌트명 등)가 출력에 모두 언급되어 있는가?
4. 인터랙션 또는 구조 설명: 호버, 애니메이션, 반응형 등 인터랙션 또는 컴포넌트 구조가 설명되어 있는가?
5. 완성도: 출력이 단순 계획이 아닌, 실제 구현 완료 상태(파일 생성 또는 전체 코드)를 나타내는가?

## 채점

- 1건 pass = 5개 기준 모두 Yes
- pass_rate = pass 건수 / 전체 실행 수
- 목표: min_pass_rate 0.8 이상
