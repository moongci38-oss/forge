# Assessment: content-creator

## 테스트 입력

각 입력에 대해 스킬을 실행하고 출력을 채점한다.

- input_1: "Write a blog post outline about AI-powered code review tools for developers"
- input_2: "Create 3 social media posts announcing a new dark mode feature"
- input_3: "Draft an email newsletter introducing a weekly dev tips series"

## 평가 기준 (Yes/No)

1. 구조화된 형식: 출력에 H2 이상의 제목(heading)이 최소 2개 이상 포함되어 있거나, 번호 목록/섹션 구분으로 명확한 구조를 갖추고 있는가?
2. 타겟 오디언스 고려: 출력에 대상 독자(예: 개발자, 팔로워, 구독자)의 관점이나 니즈, 또는 독자에 맞춘 어조/언어 수준이 명시적으로 반영되어 있는가?
3. SEO 또는 CTA 포함: 출력에 키워드 제안, 메타 설명, 해시태그, 또는 "지금 시작하기", "구독하기", "링크 클릭" 등의 명확한 행동 유도(CTA) 요소가 최소 1개 이상 포함되어 있는가?
4. 요청 콘텐츠 유형 일치: 블로그 아웃라인 요청에는 아웃라인 형식, 소셜 미디어 요청에는 개별 포스트 3개, 뉴스레터 요청에는 이메일 형식의 출력이 제공되었는가?
5. 즉시 발행 가능한 콘텐츠: 출력이 추가 편집 없이 바로 사용하거나 다음 단계로 넘길 수 있는 수준의 완성된 내용(구체적 문장, 실제 카피, 단계별 항목)을 포함하고 있는가?

## 채점

- 1건 pass = 5개 기준 모두 Yes
- pass_rate = pass 건수 / 전체 실행 수
- 목표: min_pass_rate 0.8 이상 달성
