# Video Reference Guide

게임 영상 레퍼런스를 분석하여 Unity 구현 가이드를 생성한다.

## 역할 분리

| 도구 | 역할 | 대상 |
|------|------|------|
| `/yt` | 트랜스크립트 기반 분석 (교육 영상, 강좌) | 음성 설명이 있는 YouTube 영상 |
| **이 스킬** | 프레임 기반 시각 분석 (연출, 이펙트) | 무성 게임 영상, 로컬 녹화 파일, YouTube |

## 입력

사용자가 아래 중 하나를 제공한다:

1. **로컬 영상 파일**: mp4, mov, avi, webm, mkv 경로
2. **YouTube URL**: `https://youtube.com/watch?v=xxx`
3. **분석 관점** (선택): 연출/이펙트/UI 전환/타이밍 등

## 워크플로우

### Step 1: 입력 확인

```
사용자 입력에서 추출:
- VIDEO_PATH: 영상 경로 또는 URL
- ANALYSIS_FOCUS: 분석 관점 (기본: "게임 연출 분석")
- REF_NAME: 레퍼런스 이름 (파일명 또는 사용자 지정)
```

### Step 2: 영상 분석 실행

`analyze-video.sh`를 호출하여 Gemini 프레임 분석을 실행한다.

```bash
# 프로젝트 내 저장 시
bash ~/.claude/scripts/analyze-video.sh \
  "{VIDEO_PATH}" \
  "docs/assets/video-refs/{YYYY-MM-DD}-{REF_NAME}-analysis.md" \
  "{ANALYSIS_FOCUS에 맞는 상세 프롬프트}"
```

### 분석 프롬프트 템플릿

분석 관점에 따라 프롬프트를 조정한다:

**연출 분석 (기본)**:
```
다음 게임 영상을 분석해주세요.
타임스탬프별로 연출 요소(파티클, 셰이더, 애니메이션, 사운드, UI 전환)를 식별하고,
각 요소의 시작/종료 시간, 이징, 구현에 필요한 Unity 컴포넌트를 표 형식으로 정리해주세요.

출력 형식:
| 시간 | 연출 설명 | 이펙트 요소 | 구현 가이드 |
```

**이펙트 분석**:
```
다음 게임 영상에서 파티클 이펙트와 셰이더 효과를 분석해주세요.
각 이펙트의 파라미터(색상, 크기, 수명, 속도, 블렌딩), 사용된 셰이더 기법,
Unity에서 재현하기 위한 설정값을 상세히 기술해주세요.
```

**UI 전환 분석**:
```
다음 게임 영상에서 UI 전환과 화면 전환 연출을 분석해주세요.
각 전환의 이징 함수, 지속 시간, 레이어 순서, 알파/스케일/위치 변화,
Unity UGUI/DoTween으로 구현하기 위한 코드 구조를 제안해주세요.
```

**타이밍 분석**:
```
다음 게임 영상의 연출 타이밍을 정밀 분석해주세요.
각 연출 단계의 시작/종료 시간, 딜레이, 오버랩, 시퀀스 순서를 타임라인 형태로 정리하고,
Unity DoTween Sequence 또는 Timeline으로 구현하기 위한 타이밍 차트를 생성해주세요.
```

### Step 3: 결과 구조화

분석 결과를 Spec Section 9.9에 삽입 가능한 형식으로 구조화한다:

```markdown
### 영상 레퍼런스: {레퍼런스 이름}

**원본**: `{파일 경로 또는 URL}`
**분석일**: YYYY-MM-DD

#### 타임스탬프별 연출 분석

| 시간 | 연출 설명 | 이펙트 요소 | 구현 가이드 |
|------|----------|-----------|-----------|
| 00:00-00:03 | 상자 등장 + 흔들림 | 스케일 애니메이션 + 파티클 | DoTween Sequence, Particle System |
| 00:03-00:05 | 상자 열림 + 빛 폭발 | 황금빛 파티클, 화면 플래시 | URP Bloom + Particle Burst |

#### 핵심 이펙트 목록

| 이펙트 | 유형 | Unity 구현 | 우선순위 |
|--------|------|-----------|:--------:|
| [예: 골든 파티클] | Particle System | [Burst 모드, Additive 블렌딩] | HIGH |
| [예: 화면 플래시] | Post-Processing | [URP Bloom + 화이트 오버레이] | MEDIUM |

#### 타이밍 차트

```
0.0s ─── 상자 등장 (0.5s)
0.5s ─── 흔들림 (0.3s)
0.8s ─── 대기 (0.2s)
1.0s ─── 상자 열림 (0.8s)
1.0s ─── 빛 이펙트 시작 (0.5s, 오버랩)
1.8s ─── 카드 등장 (0.6s)
2.4s ─── 결과 표시 (0.4s)
```
```

### Step 4: 결과 전달

1. 구조화된 분석 결과를 사용자에게 출력
2. `docs/assets/video-refs/`에 분석 파일 저장 (프로젝트 내 실행 시)
3. Spec 작성 중이면 Section 9.9 영상 레퍼런스 테이블에 삽입 안내

## Trine 연동

| Phase | 사용 시점 | 행동 |
|-------|----------|------|
| S3 (기획서) | 연출 기획 시 참고 영상 수집 | 영상 분석 → GDD 참고 자료 섹션에 추가 |
| Phase 2 (Spec) | 연출/이펙트 Spec 작성 시 | 영상 분석 → Section 9.9 영상 레퍼런스 테이블 삽입 |
| Phase 3 (구현) | 구현 전 상세 가이드 필요 시 | 영상 재분석 (상세 프롬프트) → 구현 참고 |

## 환경 요구사항

- `GEMINI_API_KEY` 환경변수 설정 필수
- `~/.claude/scripts/analyze-video.sh` 스크립트 존재
- Python 3 (JSON 파싱용)
- curl (API 호출용)

## 주의사항

- 영상 분석은 Gemini API 크레딧을 소비한다 — 불필요한 반복 분석 방지
- 캐싱: output-file이 이미 존재하면 API를 호출하지 않는다
- 로컬 대용량 영상(100MB+)은 업로드 시간이 길 수 있다
- YouTube URL은 Gemini가 직접 접근하므로 비공개 영상은 분석 불가

## 보안 주의사항

| 입력 유형 | 데이터 전송 방식 | 보안 주의 |
|---------|--------------|--------|
| **YouTube URL** | URL만 전달 (영상 자체 업로드 없음) | 공개 영상만 가능 — 낮은 위험 |
| **로컬 파일** | 영상 전체를 Google Files API에 업로드 | **최대 48시간 Google 서버 보관** |

**로컬 파일 분석 시 전송 금지 대상**:
- 미공개 게임플레이 영상 (발표 전 데모, 미출시 콘텐츠)
- NDA 적용 베타 테스트 영상
- 내부 플레이테스트 영상

**권장**: 미공개 콘텐츠는 이미 공개된 YouTube 레퍼런스 영상으로 대체한다.
스크립트 실행 시 로컬 파일 업로드 전 4초 확인 시간이 제공된다 (Ctrl+C로 중단 가능).
