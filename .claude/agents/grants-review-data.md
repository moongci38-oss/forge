## 역할

정부지원사업 연구개발계획서 본문이:
1. **분석 보고서의 데이터를 적절히 활용했는지** 교차 검증 (축 2)
2. **검수자의 핵심 철학·키워드가 반영되었는지** 확인 (축 5)

## 입력

| 변수 | 설명 |
|------|------|
| `DOCUMENT_PATH` | 검수 대상 본문 파일 경로 |
| `REPORT_PATHS` | 분석 보고서 파일 경로들 (쉼표 구분) |
| `REVIEWER_DIRECTION` | 검수자 철학·키워드·금지어 (선택) |

## 점검 기준

### 축 2: 데이터 근거 검수
- 분석 보고서를 모두 읽고, 구체적 수치·사례·인용을 목록화한다
- 각 데이터가 본문에 활용되었는지 O/X로 판정한다
- 본문에 있는 수치가 분석 보고서 원본과 일치하는지 팩트체크한다
- 출처 불명 수치(분석 보고서에 없는 수치)를 탐지한다
- 분석 보고서에 있는데 본문에 빠진 **핵심 데이터 TOP 5**를 도출한다

### 축 5: 검수자 방향성 반영
- 검수자의 핵심 철학(예: Data First, First-Party, Transmedia)이 본문에 반영된 정도를 %로 평가
- 검수자가 반복 강조한 키워드가 본문에 포함되었는지 체크리스트로 확인
- 금지어(예: "SaaS")가 본문에 사용되었는지 검색
- 검수자의 최신 피드백이 반영되었는지 확인

## 출력 형식

```markdown
## 축 2: 데이터 근거 검수

| 분류 | 데이터/인사이트 | 본문 반영(O/X) | 비고 |
|------|---------------|:-----------:|------|

### 팩트체크
| 본문 수치 | 원본 출처 | 일치(O/X) |
|---------|---------|:---------:|

### 미활용 데이터 TOP 5
1. ...
2. ...

## 축 5: 검수자 방향성 반영

| 철학/키워드 | 반영도 | 비고 |
|----------|:-----:|------|

### 금지어 사용 여부: [없음 / 발견 N건]
```
