# 성공은 침묵 (Success Is Silent)

## 규칙

**도구 실행 성공 시 불필요한 출력 생성 금지.** 결과가 짧고 명확하면 요약도 하지 말 것.

## 적용 범위

| 상황 | 올바른 행동 |
|------|------------|
| Bash 명령 성공 (exit 0) | 결과가 짧으면 그대로. 2000자 초과면 핵심만 |
| 파일 쓰기/편집 성공 | "완료" 한 줄 or 침묵. 전체 내용 반복 금지 |
| 테스트 통과 | 통과 사실만. 각 테스트 케이스 열거 금지 |
| 훅 실행 성공 | stdout 없음 (PostToolUse offload 훅 참조) |

## 이유

- 성공 로그가 컨텍스트 윈도우를 채우면 정확도 저하 (Context Rot)
- 50턴 후 컨텍스트 오염이 초반 대비 응답 품질 유의미하게 감소
- `~/.claude/offload/` 훅이 대용량 출력 자동 아카이브 → 필요 시 Read 가능

## 예외 (침묵 금지)

- 실패(non-zero exit, error) → 전체 에러 메시지 반드시 출력
- 보안 경고 → 항상 명시
- 파괴적 작업 확인 요청 → 상세히 설명
- 사용자가 명시적으로 "보여줘" 요청 → 보여줌
