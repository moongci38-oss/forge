# Grants (정부과제) 규칙

## 자료 우선순위
P1(검수자 방향성 + 작성요령) > P2(분석 문서) > P3(학습 자료)
근거 없는 내용 생성 절대 금지.

## 작성 순서
분석 보고서 요약 먼저가 아님.
검수자 방향성 문서의 기술 체계 파악 → 근거 끼워 넣기 순서로 작성.

## 삭제 금지
목차별 평가 기준(대섹션)과 세부목차별 작성요령(파란색) 절대 삭제 금지.

## R2 팩트 체크
반복 발견 오류 3종 자동 감지:
1. 수치·통계 출처 미표기
2. 사업명·기관명 오기
3. 연도·기간 불일치

## 검수자 피드백
검수자(윤효성) 피드백 → 참고자료 폴더 채팅피드백 파일에 누적 업데이트 + 분석 보고서에도 반영

## 색상 컨벤션
- 파란색 = 작성요령 출처 태그
- 빨간색 = 강조/경고

## 산출물 경로
- grants-review 리포트 → `_prompts/reviews/` 폴더 전용. handover 폴더 금지.
- grants-write/review 스킬 → 전역 아님. `forge-outputs/09-grants/{project}/.claude/skills/`에 위치.

## 도식 금지
본문에 Mermaid raw 코드 블록 직접 삽입 절대 금지.
figures-design/*.md 컨셉 SSoT + figures/*.jpg 마커가 유일한 도식 워크플로우.

## figures-design
모든 이미지는 `figures-design/*.md` 정의 파일이 먼저 존재해야 함.
이미지 수정·재생성·검증 전에 항상 정의 md 먼저 만들거나 수정. 예외 없음.
