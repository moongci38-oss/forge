# 에이전트 반환 형식 표준 (Agent Response Format)

> Sub-agent 결과물이 상위 context를 오염시키지 않도록 반환 형식을 표준화한다.

---

## 반환 봉투 (Response Envelope)

```markdown
## Result
(1-3문장 — 무엇을 완료했는지)

## Key Findings / Output
(구조화 데이터: 테이블, 불릿, 코드 블록)

## Decisions Made
(비자명한 선택과 그 이유만 — 자명한 것은 생략)

## Blockers / Questions
(상위/사용자 주의 필요 사항 — 없으면 생략)
```

## 토큰 예산

| 에이전트 유형 | 목표 | 상한 |
|-------------|------|------|
| 탐색/검색 (Explore) | 300-500 토큰 | 1,000 |
| 분석/리서치 | 500-1,000 토큰 | 2,000 |
| 구현/코드 생성 | 코드 + 200단어 요약 | 제한 없음 |

## Context 격리 원칙

- 에이전트 출력은 **자기완결적(self-contained)** — "위에서 논의한 대로", "앞서 언급한" 등 상위 context 참조 금지
- 파일 내용을 **통째로 반환 금지** — 요약 또는 변경 diff만 반환
- 에러 스택 트레이스 전문 금지 — `에러 메시지 + file:line` 만 반환
- Raw API 응답 금지 — 파싱된 관련 필드만 반환

## 민감 정보 필터링

- `.env` 값, 토큰, 비밀번호 반환 금지
- 파일 경로에 `06-finance/`, `07-legal/`, `08-admin/` 포함 시 내용 반환 금지
- 사용자 개인정보(이메일, 전화번호) 반환 시 마스킹
