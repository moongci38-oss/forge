# Design Rules — Forge 파이프라인 레이아웃/UX 원칙

> 토큰(색상/타이포/스페이싱)은 `instagram-default.json` 참조.
> 이 파일은 레이아웃 구조, 구성 원칙, 금지사항을 정의한다.
> SDD Check C (code-reviewer) + PGE Phase 1 Planner + PGE Phase 3 QA 트랙 B에서 참조.

---

## 1. 색상 사용 원칙

- **단일 CTA color**: `#0095F6` — 활성 상태, 주요 버튼에만 사용. 배경 전체에 칠하지 않는다
- **순수 검정(#000000) 사용 금지** — 가장 어두운 텍스트는 `#262626` (text.primary)
- **텍스트 계층 5단계**: `#262626` → `#4A4A4A` → `#8E8E8E` → `#C7C7C7` → `#EFEFEF`
- **배경**: 흰 배경 `#FFFFFF`, 서브 배경 `#FAFAFA`, 구분 배경 `#EFEFEF`
- **에러**: `#ED4956` / **성공**: `#00C853` — 상태 표시에만 사용
- **브랜드 그라데이션**: `linear-gradient(45deg, #FEDA75, #FA7E1E, #D62976, #962FBF, #4F5BD5)` — 장식용, 강조 포인트에만

---

## 2. 타이포그래피 원칙

- **숫자와 단위 비율 2:1**: 숫자 크기가 단위 텍스트의 2배 (예: 숫자 28px + 단위 14px)
- **통화/수치 축약**: `1,200,000` → `1.2M`, `48,000` → `48K` — `whitespace-nowrap` 필수
- **폰트 스택**: `"SF Pro Display", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- **행간**: 본문 `line-height: 1.5`, 제목 `line-height: 1.2`
- **자간**: `-0.02em` (SF Pro 기준)

---

## 3. 레이아웃 섹션 4타입

페이지는 4가지 섹션 타입만 사용하여 구성한다.

| 타입 | 형태 | 용도 |
|------|------|------|
| **Type A — 풀 카드** | 단일 카드, 전체 너비 또는 대형 | 핵심 지표, 히어로, 상세 정보 |
| **Type B — 그리드** | 2~4열 동일 크기 카드 | 대시보드 요약, 항목 목록 |
| **Type C — 캐러셀/스크롤** | 가로 스크롤 카드 열 | 연관 항목, 추천, 탐색 |
| **Type D — 히어로 + 보조** | 대형 1개 + 소형 2~3개 병렬 | 랜딩, 주요 기능 소개 |

**리듬 규칙**:
- 같은 섹션 타입 연속 2개 이상 배치 금지 (Type B 다음 Type B 금지)
- 긴 섹션(높은 콘텐츠) 다음에는 짧은 섹션 배치
- 정보 밀도는 페이지 아래로 갈수록 증가

---

## 4. 카드 구조 원칙

- **모든 콘텐츠는 카드 안에** — 카드 밖에 단독 텍스트/버튼/입력 필드 배치 금지
- 카드 border-radius: `8px` (md) 또는 `16px` (lg)
- 카드 내부 padding: 최소 `16px` (`lg`)
- 카드 간격: `12px` (`md`) 또는 `16px` (`lg`)

---

## 5. 절대 금지사항 (7개)

1. **CTA 색상(`#0095F6`)으로 카드 배경 칠하기** — 활성/클릭 상태에만 사용
2. **순수 검정(`#000000`) 사용** — `#262626`으로 대체
3. **섹션 사이 구분선(border/hr)** — 섹션 타입과 여백으로 구분
4. **카드 밖에 콘텐츠 배치** — 항상 카드 래퍼 안에
5. **카드 내부에 CTA 버튼 + 입력 필드 동시 배치** — 역할 혼재 금지
6. **임시/플레이스홀더 컴포넌트로 마무리** — 모든 컴포넌트는 실제 동작해야 함
7. **같은 섹션 타입 연속 배치** — 레이아웃 리듬 유지

---

## 6. 인터랙션 원칙

- **터치 타겟 최소 44×44px** — 모든 클릭/탭 가능 요소
- **포커스 링 필수** — `outline: none` 단독 사용 금지, 반드시 대체 스타일 제공
- **상태 전환**: `transition-colors duration-200` 기본, hover/focus/active 3가지 상태 필수
- **로딩/에러/빈 상태 3가지** — 데이터 fetch가 있는 모든 컴포넌트에 필수

---

## 7. 간격 시스템

4px 배수 기준 (Tailwind spacing):

| 토큰 | 값 | 용도 |
|------|-----|------|
| `xs` / `p-1` | 4px | 아이콘 내부, 태그 |
| `sm` / `p-2` | 8px | 인라인 요소 간격 |
| `md` / `p-3` | 12px | 카드 내 요소 간격 |
| `lg` / `p-4` | 16px | 카드 패딩, 섹션 간격 |
| `xl` / `p-6` | 24px | 섹션 패딩, 대형 간격 |

홀수 px 값(5px, 7px, 13px 등) 사용 금지.

---

## 8. 반응형 원칙

- **3 breakpoint 기준**: 모바일(~640px) / 태블릿(641~1024px) / 데스크톱(1025px+)
- 고정 `width: Npx` 레이아웃 컨테이너 금지 → `max-width` + `w-full` 사용
- 테이블은 `overflow-x-auto` 래퍼 필수
