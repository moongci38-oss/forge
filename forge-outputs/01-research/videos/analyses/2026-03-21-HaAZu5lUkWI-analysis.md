## TL;DR

안드레 카르파티의 오픈소스 AutoResearch를 Claude Code와 결합해, 콜드 이메일 답장률 등 비즈니스 지표를 AI가 스스로 A/B 테스트하며 최적화하는 "자기 개선 파이프라인"을 GitHub Actions 기반으로 구축하는 방법을 실연한다.

## 핵심 포인트

1. **AutoResearch = AI가 스스로 실험하는 파이프라인** — 카르파티가 공개한 오픈소스 레포. 에이전트에게 학습 환경을 주고 밤새 자율 실험하게 두면, 아침에 더 나은 모델과 실험 로그가 준비된다. [🕐 00:17](https://youtu.be/HaAZu5lUkWI?t=17)

2. **핵심 원칙: 추적 가능한 지표 + 수정 가능한 변수** — 이 두 가지만 있으면 어떤 비즈니스 영역에도 AutoResearch 원칙을 적용할 수 있다. [🕐 02:16](https://youtu.be/HaAZu5lUkWI?t=136)

3. **실제 사례 — 콜드 이메일 답장률 자동 최적화** — Instantly API로 답장률을 지표로 삼고, AI가 이메일 카피 변형을 생성·배포·평가하는 루프를 GitHub Actions 크론(4시간)으로 자동화. 1차 테스트 1.5% → 12차 테스트 2%+ 상승 확인. [🕐 01:36](https://youtu.be/HaAZu5lUkWI?t=96)

4. **인간 대비 AI의 실질적 우위 — 규모와 속도** — 인간이 하루 몇 번 최적화할 때, AI는 매시간 24회 루프 실행. 개별 판단 품질이 아니라 반복 속도의 차이가 누적 성과를 결정한다. [🕐 06:47](https://youtu.be/HaAZu5lUkWI?t=407)

5. **적용 가능한 비즈니스 영역 목록** — 콜드 이메일 카피, 랜딩 페이지 전환율, 광고 소재, 챗봇 고객 만족도, 이커머스 상품 설명, 유튜브 제목, 뉴스레터 제목 줄, 가격 페이지 — 지표와 API만 있으면 전부 자동화 가능. [🕐 07:37](https://youtu.be/HaAZu5lUkWI?t=457)

6. **구현 3단계** — (1) AutoResearch 레포 클론 → (2) 목표·지표·테스트 방법 정의 파일 작성 → (3) GitHub Actions 크론으로 클라우드에 스케줄링. [🕐 11:01](https://youtu.be/HaAZu5lUkWI?t=661)

7. **오케스트레이터-서브에이전트 구조** — 최상위 오케스트레이터가 카피 생성 서브에이전트, Instantly API 호출, 결과 저장을 조율하는 멀티에이전트 아키텍처로 구성. 코드는 Claude Code가 전부 자동 생성. [🕐 15:50](https://youtu.be/HaAZu5lUkWI?t=950)

8. **GitHub Actions 3단계 워크플로** — 수집(이전 실험 결과) → 생성(새 챌린저 카피) → 배포(캠페인 활성화). 슬랙 웹훅으로 변경 시 알림. [🕐 17:52](https://youtu.be/HaAZu5lUkWI?t=1072)

9. **카르파티 프롬프트 품질 주의** — 카르파티 본인도 "내 프롬프트는 형편없고 더 나은 것 만들기 매우 쉬울 것"이라 명시. 레포는 구조 참고용이지 프롬프트 품질 기준으로 삼으면 안 된다. [🕐 12:06](https://youtu.be/HaAZu5lUkWI?t=726)

10. **Whisper Flow 활용 — 음성 프롬프팅** — 발표자가 Claude Code에 긴 지시를 줄 때 음성 받아쓰기 도구(Whisper Flow)로 자연어를 그대로 입력. 타이핑 대신 구술로 에이전트 지시가 가능함을 시연. [🕐 13:50](https://youtu.be/HaAZu5lUkWI?t=830)

## 비판적 분석

### 주장 1: "AI 에이전트가 최적화를 반복할수록 지표(답장률)가 지속적으로 향상된다"
- **제시된 근거**: 1차 테스트 1.5% → 12차 테스트 2%+ 상승 수치 제시
- **근거 유형**: 경험적 관찰 (단일 사례, 샘플 크기 미공개)
- **한계**: 12회 반복의 전체 데이터 미공개. 통계적 유의성 불확인. 콜드 이메일 답장률은 대상 리스트 품질, 발신 도메인 평판, 계절성 등 외부 변수 영향이 크다. 수렴 한계(최적점 이후 개선 둔화)에 대한 언급 없음.
- **반론/대안**: 순수 카피 최적화 효과인지 다른 요인(더 나은 리드 리스트, 발신 타이밍 변화)과 혼재 가능성 있음. 진정한 A/B 테스트라면 동일 모집단에 동시 배포 후 비교해야 인과관계 확립 가능.

### 주장 2: "측정 가능한 지표와 API만 있으면 어떤 비즈니스 영역에도 적용 가능하다"
- **제시된 근거**: 콜드 이메일, 랜딩 페이지, 광고 소재, 챗봇 스크립트, 이커머스 상품 설명, 유튜브 제목 등 8개 이상 예시 나열
- **근거 유형**: 논리적 추론 (실제 적용 확인은 콜드 이메일 한 가지)
- **한계**: 나열된 사례 중 실제 구현을 시연한 것은 콜드 이메일뿐. 랜딩 페이지 전환율은 피드백 루프가 느려(방문자 수 확보 필요) 시간당 반복이 무의미할 수 있음. 고객 만족도 점수는 수집 방식과 시차 문제가 있음.
- **반론/대안**: 피드백 루프 속도(분 단위 vs 일 단위)에 따라 자동화 효익이 크게 달라진다. 지표 지연이 긴 영역(SEO, 브랜드 인지도)에는 이 패러다임이 적합하지 않음.

### 주장 3: "인간이 개입하지 않아도 AI가 더 나은 판단을 한다"
- **제시된 근거**: "AI 모델보다 내가 더 나은 결정을 내렸겠지만 그건 중요하지 않다. 내가 최적화에 쓸 수 있는 시간이 AI보다 훨씬 적기 때문" (발표자 직접 발언)
- **근거 유형**: 의견 (합리적 추론이나 실증 데이터 아님)
- **한계**: "품질보다 속도" 논리는 최적화 대상이 단순 텍스트 변형일 때만 성립. 브랜드 일관성, 법적 리스크, 오해 소지가 있는 표현 등은 인간 검수가 필요한 영역. 24/7 자동 이메일 발송은 스팸 도메인 페널티 위험도 있음.
- **반론/대안**: 인간-AI 협업 모델 — AI가 변형 생성+지표 수집을 담당하고 인간이 주기적으로 방향성 검수하는 hybrid 루프가 순수 자동화보다 안전하고 현실적.

### 주장 4: "Claude Code가 모든 코드를 자동 생성했다"
- **제시된 근거**: "저는 하나도 작성하지 않았습니다. 이 모든 것은 카르파티가 한 것과 매우 유사한 논리를 따릅니다" (발표자 발언)
- **근거 유형**: 경험적 주장
- **한계**: 생성된 코드의 실제 실행 결과(오류 없이 동작하는지, 프로덕션 수준인지)를 영상 내에서 검증하지 않음. 데모 목적의 코드와 실제 프로덕션 코드 사이의 간극이 존재할 수 있음.
- **반론/대안**: 코드 생성 자체보다 "실제로 Instantly API와 연동되어 이메일이 발송됐는가"의 end-to-end 검증이 더 중요한 증거가 될 것.

## 실행 가능 항목

### Forge 파이프라인 적용

- [ ] **AutoResearch 레포 분석** — `github.com/karpathy/autoresearch` 구조를 읽고 프롬프트 파일(program.md)과 실험 루프 로직을 Forge 파이프라인에 통합 가능한지 검토
- [ ] **Forge 스킬 자동 개선 파이프라인 설계** — 스킬 성능 지표(예: 출력 품질 점수, 토큰 효율성)를 정의하고 AutoResearch 원칙으로 스킬 프롬프트를 자동 변형·평가하는 루프 설계
- [ ] **GitHub Actions 크론 템플릿 추가** — `forge/dev/templates/` 또는 `forge/shared/` 에 AutoResearch 스타일 크론 워크플로 템플릿 추가 (수집→생성→배포 3단계 구조)
- [ ] **Whisper Flow 활용 검토** — Claude Code 사용 시 음성 받아쓰기로 긴 지시 입력 효율화 가능 여부 확인

### Portfolio 프로젝트 적용

- [ ] **랜딩 페이지 자동 A/B 테스트** — Portfolio 랜딩 페이지 전환율을 지표로 설정, AutoResearch 루프로 카피 변형 자동화 (단, 피드백 루프 지연 고려 — 트래픽 확보 후 적용)
- [ ] **블로그 제목 자동 최적화** — YouTube Data API v3 대신 Analytics API로 블로그 CTR 수집 → 제목 변형 자동 실험 파이프라인 구성
- [ ] **뉴스레터 제목 줄 실험** — 뉴스레터 발행 시 제목 A/B 테스트를 AutoResearch 루프로 자동화, 오픈율 지표 추적

### GodBlade 적용

- [ ] **게임 UI 텍스트 최적화** — 튜토리얼 안내 텍스트, 스킬 설명 등을 지표(유저 이탈률, 클리어율)로 자동 변형 실험 가능성 검토
- [ ] **프롬프트 엔지니어링 자동화** — GodBlade 내 AI 시스템(전투 AI, 대사 생성 등)의 프롬프트를 AutoResearch 원칙으로 자율 개선 — 단, 게임 내 측정 가능한 지표 설계가 선행되어야 함
