# 비교 분석 리포트: Claude Code 고급 토큰 절약 기법 vs 우리 시스템
> 영상: 캐슬 AI — 클로드 코드 맥스 요금제 쓰지 마세요 (3부 고급편) | 분석일: 2026-04-09

---

## 개요

영상에서 소개된 고급 기법 10개를 우리 Forge 시스템과 비교 분석한다.
자막 신뢰도 Low(자동생성)로 일부 기술 용어 오기가 있어 외부 검색으로 교차 검증 완료.

## 이미 잘 하고 있는 것

1. **멀티 터미널 + Worktree**: tmux 기반 멀티 세션 운영 + 동시 수정 시 worktree 사용. 영상 권장 방식과 동일
2. **에이전트 역할 분리**: 스킬 시스템(60개+)으로 역할별 에이전트 구조화. 도구 수준 분리까지는 미완
3. **미사용 MCP 정리**: forge-core.md에서 불필요 MCP 정리 권고 이미 포함
4. **프롬프트 캐싱**: DISABLE_PROMPT_CACHING 미설정 = 기본값 활성. 정상 상태

## 실질적 갭 — 우선순위별

### GAP-1: 환경변수 명시적 제어 미설정 (즉시 해결 가능)
- `DISABLE_NON_ESSENTIAL_MODEL_CALLS=1` 미설정
- 영향: 불필요한 백그라운드 API 호출 발생 가능
- 해결: `forge/.claude/settings.json` env 섹션에 1줄 추가

### GAP-2: read-once 훅 미적용 (1~2시간)
- 세션 내 동일 파일 반복 읽기 비용 발생
- 실측: 세션당 약 40% 읽기 토큰 절감 (dev.to 실측 데이터)
- 우리 시스템 맥락: Forge 세션에서 CLAUDE.md, rules/*.md, forge-workspace.json 등 반복 읽기 다수
- 해결: 외부 스크립트 검토 후 설치 OR bash+jq로 자체 구현

### GAP-3: 로그 전처리 훅 미구현 (1~2시간 자체 구현)
- 현재 agent-token-budget.sh가 오류 패턴 감지는 하지만 Claude에게 전달되는 출력 필터링은 없음
- 해결: Bash PostToolUse 훅에 FAILED/ERROR 라인 필터 추가

### GAP-4: MCP 도구 최적화 (P2, 이번 달)
- 전역 4개 + Forge 2개 MCP 운영 중. 도구 설명 기본값 그대로
- MindStudio 기준 서버당 최대 18,000 토큰/턴 오버헤드
- 우선 대상: nano-banana (직접 접근 가능한 MCP)

### GAP-5: QMD (P2, 검토 후 결정)
- Forge는 파일 구조가 비교적 단순하여 즉각적 필요성 낮음
- GodBlade Unity 코드베이스(`.cs` 파일 다수)에서 효과 높을 가능성

## 결론

영상은 고급 최적화 기법을 잘 정리하고 있으나, 수치 인용의 출처 불명확과 자막 오기(QMD→"QMD", CCUCG→ccusage 등) 주의 필요. 우리 시스템에서 **즉시 적용 가치가 있는 것은 환경변수 1개 설정(P0)과 read-once 훅 도입(P1)** 이다. QMD와 MCP 최적화는 현재 blocking 없이 장기 개선 과제.
