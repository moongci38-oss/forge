# 하네스 개선 마스터플랜
> 작성일: 2026-04-06 | 출발점: YouTube 3건 분석 + GodBlade 실전 피드백

## 배경

YouTube 영상 분석(하네스 엔지니어링, 바이브코딩, PPTX 자동화)에서 도출된 개선 사항과,
GodBlade 실전 개발에서 발견된 **반복 실패 패턴**을 종합하여 마스터플랜을 수립한다.

**핵심 문제**: 에이전트가 실수를 반복하고, 컨텍스트를 잃고, 기존 소스를 제대로 분석하지 못한다.

## Phase 2 — GodBlade 하네스 강화 (우선순위 최상)

> GodBlade 실전 개발에서 발견된 3대 문제 해결

### 2-1. 인수인계 문서 → AI 컨텍스트 문서화 (P0)

**문제**: 에이전트가 프로젝트 구조를 모르고 코드만 보고 수정 → 의존성 놓침 → 반복 실패

**작업 내용**:
- GodBlade 인수인계 문서를 CLAUDE.md 또는 `.claude/reference/` 하위 문서로 구조화
- 포함할 내용:
  - **UI 시스템 의존성 맵**: 어떤 스크립트가 어떤 UI 컴포넌트를 제어하는지
  - **이벤트 흐름도**: 특정 이벤트 후 어떤 초기화/리셋 로직이 실행되는지
  - **건드리면 안 되는 것들**: 수정 시 사이드이펙트가 큰 핵심 시스템 목록
  - **이펙트/연출 시스템 구조**: DOTween, Animator, ParticleSystem 사용 패턴
  - **Inspector 설정 의존성**: 코드에 안 보이지만 Inspector에서 설정된 값들

**기대 효과**: 매 세션마다 소스 전체를 읽을 필요 없이 문서 참조로 컨텍스트 절약

**소요**: 반나절~1일 (사용자와 함께 작성)

### 2-2. 수정 전 의존성 분석 필수 규칙 (P0) ✅ 완료

**문제**: 버튼 크기 수정 → 다른 스크립트/애니메이션이 덮어씀 → 원복됨

**작업 내용**:
- GodBlade `.claude/rules/`에 `pre-modification-analysis.md` 규칙 추가
- 규칙 내용:
  ```
  Unity UI/연출 수정 전 필수 절차:
  1. 대상 컴포넌트의 RectTransform/값을 참조하는 모든 스크립트 검색
  2. DOTween/Animator에서 같은 프로퍼티를 제어하는지 확인
  3. 이벤트 핸들러(OnEnable, OnDisable, 커스텀 이벤트)에서 초기화 로직 확인
  4. LayoutGroup 등 자동 레이아웃 영향 확인
  → 의존성 맵 작성 후에만 수정 시작
  ```

**기대 효과**: "값만 바꾸면 되겠지" → "이 값을 건드리는 곳이 3군데 더 있음" 사전 파악

**소요**: 30분

### 2-3. 작업 단위 세분화 + 파일 기반 컨텍스트 (P0) ✅ 완료

**문제**: 컨텍스트 소모가 커서 중간에 맥락 소실 → 같은 분석 반복

**작업 내용**:
- GodBlade `.claude/rules/`에 `context-management.md` 규칙 추가
- 규칙 내용:
  ```
  컨텍스트 관리 원칙:
  1. 한 세션에 버그 1개 또는 기능 1개만 처리
  2. 분석 결과는 반드시 파일로 저장 후 참조 (.claude/state/current-analysis.md)
  3. 수정 시도 실패 시 실패 원인을 learnings.jsonl에 즉시 기록
  4. 대규모 소스 읽기보다 Grep 타겟팅 우선
  ```

**기대 효과**: 컨텍스트 소실 시에도 파일에 분석 결과가 남아있어 복구 가능

**소요**: 30분

### 2-4. 영상 레퍼런스 → 구현 명세 파이프라인 (P1) ✅ 완료

**문제**: 영상을 주고 "이대로 구현해줘" → 20% 수준 구현 → 세부사항 누락

**작업 내용**:
- `/video-reference-guide` 스킬로 영상을 프레임 단위 분석
- 분석 결과를 구현 명세(타이밍, 이징 곡선, 파티클 파라미터 등)로 변환
- PGE Evaluator에 영상 대비 체크리스트 루브릭 추가

**워크플로우**:
```
영상 → /video-reference-guide → 구현 명세.md → PGE(명세 기반) → 구현 → 검증
```

**기대 효과**: "대충 비슷하게"가 아닌 "명세의 각 항목 충족 여부" 기준 평가

**소요**: 스킬 이미 존재, 워크플로우 정립만 필요 (1시간)

### 2-5. Unity 빌드 체크 hook (P1) ✅ 완료

**문제**: 에이전트가 "수정 완료"라고 하지만 빌드 에러 또는 런타임 에러 존재

**작업 내용**:
- GodBlade `.claude/hooks/`에 `post-write-build-check.sh` 추가
- Edit/Write 후 `msbuild` 빌드 체크 실행 (컴파일 에러 즉시 피드백)
- Unity MCP가 제공하는 컴파일 상태 확인 연동 검토

**소요**: 반나절

### 2-6. 디버그 로그 삽입 전략 규칙 (P1) ✅ 완료 (2-3에 통합)

**문제**: Unity MCP로 런타임 동작(이벤트 후 어떤 스크립트가 값을 덮어쓰는지) 추적 어려움

**작업 내용**:
- 규칙: "원인 불명의 값 변경 문제 시, 해당 프로퍼티를 set하는 모든 곳에 Debug.Log 삽입 → 플레이 → 로그로 원인 특정"
- /investigate 스킬의 GodBlade 특화 가이드에 이 전략 포함

**소요**: 30분
