# 에이전트 회의 결과 (Competing Hypotheses)

> S3 기획서(PRD/GDD) 최상단에 배치. 검토자가 의사결정 과정을 확인할 수 있게 한다.

## S2 기획 디렉션 참조

> pipeline-orchestrator가 S2 gate-log에서 자동 주입

```
1. 전략 방향: {한 문장}
2. 경험 원칙: {한 문장}
3. 범위 경계: Do {N}개 / Don't {N}개
4. 품질 기준: {한 문장}
5. 벤치마크: {레퍼런스}
```

## 참가 에이전트

| 에이전트 | 역할/관점 | 초안 |
|---------|---------|------|
| {에이전트A} | {관점} | Draft A |
| {에이전트B} | {관점} | Draft B |

## 비교 분석

> 최소 5개 항목 비교. 각 항목에 구체적 차이점과 장단점을 기술한다.

| 비교 항목 | Draft A | Draft B | 채택 | 선택 이유 |
|----------|---------|---------|:----:|----------|
| 핵심 아키텍처 | {A 접근 + 장단점} | {B 접근 + 장단점} | A/B | {구체적 근거} |
| 유저 플로우 | {A 접근 + 장단점} | {B 접근 + 장단점} | A/B | {구체적 근거} |
| 기술 스택 | {A 접근 + 장단점} | {B 접근 + 장단점} | A/B | {구체적 근거} |
| 수익화/비즈니스 모델 | {A 접근 + 장단점} | {B 접근 + 장단점} | A/B | {구체적 근거} |
| 리스크/확장성 | {A 접근 + 장단점} | {B 접근 + 장단점} | A/B | {구체적 근거} |
| {추가 항목} | {A 접근} | {B 접근} | A/B | {근거} |

## S2 디렉션 탈락 필터

> S2 기획 디렉션의 범위 경계 Don't 태그와 대조하여 위반 초안을 제거한다.

### Don't 태그 대조 결과

| Don't 태그 | Draft A | Draft B | 비고 |
|-----------|:-------:|:-------:|------|
| `{tag-1}` | ✅ 미위반 | ✅ 미위반 | |
| `{tag-2}` | ✅ 미위반 | ❌ 위반 | {위반 내용 설명} |

### 탈락 판정

- **Draft A**: {PASS / FAIL — Don't `{태그}` 위반}
- **Draft B**: {PASS / FAIL — Don't `{태그}` 위반}

> 탈락된 초안은 이하 비교 대상에서 제외한다. 전체 초안이 탈락한 경우 **[STOP]** Human 에스컬레이션.

## 디렉션 충돌 처리

> Don't 항목이 기술적으로 불가피하게 필요한 경우 여기에 기록한다.

| 충돌 Don't 태그 | 기술적 필요 사유 | 제안 | Human 결정 |
|---------------|---------------|------|:----------:|
| `{tag}` | {왜 기술적으로 필수인지} | {대안 A / 대안 B / Don't 해제} | ⬜ 대기 |

## 최종 결정

- **선택 근거**: {구체적 이유}
- **보완 요소**: {다른 Draft에서 가져온 요소}
